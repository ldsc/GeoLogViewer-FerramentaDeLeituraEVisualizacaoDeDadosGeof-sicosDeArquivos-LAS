\section{Etapas\label{sec: Etapas}}

Esta proposta, caso aprovada, será desenvolvida seguindo as etapas
abaixo descritas. Um detalhamento das etapas esta disponível \href{https://github.com/ldsc/ProjetoEngenharia-0-Metodologia-Instrucoes-Etapas-ModeloProfessorBueno}{aqui}:
\begin{itemize}
\item Etapa 0 - Definição do desafio tecnológico
\begin{itemize}
\item Identificação do problema, descrição do desafio tecnológico, objetivo
geral e solução esperada (TRL/CRL).
\end{itemize}
\item Etapa 1 - Elaboração do pré-projeto
\begin{itemize}
\item Elaboração da primeira versão do projeto, rascunho inicial.
\end{itemize}
\item Etapa 2 - Elaboração do projeto - detalhamento e contrato (este documento)
\begin{itemize}
\item Após aprovação do pré-projeto os alunos detalham o mesmo gerando o
projeto. Os dados do pré-projeto podem ser copiados para o projeto,
a seguir tudo deve ser detalhado. As etapas associadas devem ser bem
definidas e suas atividades (note que para cada etapa teremos diversas
atividades). O cronograma simplificado deve ter as etapas e prazos,
deve ser realista. O orçamento deve ser bem realizado.
\end{itemize}
\item Etapa 3 - Modelagem de engenharia: 
\begin{itemize}
\item Concepção; Elaboração; Análise Orientada a Objeto; Projeto do Sistema;
Projeto Orientado a Objeto.
\item Realização de testes lógicos; Documentação (geração dos documentos
de modelagem e diagramas associados).
\item Você encontra diagramas associados a esta etapa nos apêndices e \href{https://sites.google.com/view/professorandreduartebueno/ensino/introdu\%25C3\%25A7\%25C3\%25A3o-ao-projeto-de-engenharia?authuser=0\#h.jzh3wtardi2q}{no site da disciplina (etapa modelagem)}.
\end{itemize}
\item Etapa 4 - Ciclos de planejamento, detalhamento e construção/implementação:

\begin{itemize}
\item Esta etapa é dividida em ciclos de planejamento, detalhamento e construção/implementação,
pense nisso como uma divisão temporal do trabalho e das entregas dos
sub-produtos (partes do produto final).
\item Nesta etapa, a mais demorada do projeto, temos:
\begin{itemize}
\item Reunião no início do ciclo:
\begin{itemize}
\item A equipe faz uma reunião onde selecionam as \emph{features/funcionalidades
}a serem implementadas.
\end{itemize}
\item Reuniões diárias:
\begin{itemize}
\item Fazem reuniões diárias, de 15 minutos, onde falam o que fizeram no
dia anterior e o que irão fazer no dia.
\item Na sexta-feira fazem um balanço do que foi feito.
\end{itemize}
\item Reunião no final do ciclo
\begin{itemize}
\item No final do ciclo entregam o sub-produto que deve estar funcional.
\end{itemize}
\end{itemize}
\item O último ciclo é o ciclo de fechamento, o mesmo inclui:
\begin{itemize}
\item Realização de testes de integração e confecção dos manuais do desenvolvedor
e do usuário.
\item Os manuais devem ter vários testes do sistema rodando (com todas as
informaç\~{e}os para executar/testar o produto desenvolvido).
\end{itemize}
\item Você encontra diagramas associados a esta etapa nos apêndices e \href{https://sites.google.com/view/professorandreduartebueno/ensino/introdu\%25C3\%25A7\%25C3\%25A3o-ao-projeto-de-engenharia?authuser=0\#h.mwce3hjuxu1n}{no site da disciplina (etapa planejamento/detalhamento/construção)}.
\end{itemize}
\item Etapa 5 - Entrega do produto: 
\begin{itemize}
\item Verificações finais na documentação e testes.
\item Apresentação do produto.
\item Entrega do produto.
\end{itemize}
\end{itemize}
